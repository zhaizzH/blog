# 制作 Windows + Linux 双系统

## 前言

本教程指导你在同一块移动固态硬盘上同时安装 Windows 和 Linux 两个系统，并划分独立的存储分区用于数据共享，最终硬盘的分区布局如下：

|  | EFI | Windows 分区 | Linux 分区 | 存储分区 |
| ---- | --- | ------------ | ---------- | -------- |
| 大小 | 550MB | 约 100GB | 约 50GB | 约 328GB |
| 文件系统 | FAT32 | NTFS | ext4 | exFAT |

制作分为三大步骤：

1. **第一步**：使用 WinToGo 工具安装 Windows 系统 (参考 [制作 WinToGo](/guide/wintogo))
2. **第二步**：通过 Ventoy 安装 Linux Mint 到剩余空间
3. **第三步**：使用 DiskGenius 在最后的剩余空间中创建存储分区

::: warning 注意
本教程以 **512GB 移动固态硬盘** 为例进行演示，如果你的硬盘容量不同，可根据实际情况调整各分区大小
:::

## 准备工作

### 硬件要求

| 项目 | 最低要求 | 推荐配置 |
| ---- | -------- | -------- |
| 移动存储设备 | USB 3.0 移动固态硬盘(128GB+) | USB 3.0/3.1 移动固态硬盘(256GB+) |
| 读取速度 | 50MB/s | 150MB/s 以上 |
| 写入速度 | 20MB/s | 50MB/s 以上 |
| 一个空闲 U 盘 | 8GB+(用于 Ventoy 启动盘) | 16GB+ |

::: warning 注意
同时容纳 Windows 和 Linux 两个系统，建议移动硬盘容量不低于 **128GB**，推荐 **256GB** 以上
:::

### 软件准备

- Windows 10/11 ISO 镜像
- WinToGo 制作工具 (参考 [制作 WinToGo](/guide/wintogo))
- Linux Mint ISO 镜像
- Ventoy 启动盘制作工具
- [DiskGenius](https://www.diskgenius.cn/)
- 一台正常运行的 Windows 电脑

从 Linux Mint 官网下载最新稳定版：

- [Linux Mint 官网下载](https://linuxmint.com/download.php)

| 版本 | 桌面环境 | 适合人群 |
| ---- | -------- | -------- |
| **Cinnamon** | Cinnamon (推荐) | 功能最全，适合大多数用户 |
| **MATE** | MATE | 老旧硬件,资源占用更低 |

## 第一步:制作 WinToGo

::: info 提示
详细的 WinToGo 制作步骤请参考 [制作 WinToGo](/guide/wintogo)，此处仅列出关键步骤
:::

### 1. 制作 WinToGo 系统

按照 [制作 WinToGo](/guide/wintogo) 的教程，将 Windows 系统安装到移动硬盘上

### 2. 设置 Windows 分区大小

在 WTG 工具的 **"高级选项"** 中，将 Windows 分区大小设置为 **102400MB** (约 100GB)，剩余空间后续用于 Linux 和存储分区

::: info 提示
Windows 分区建议保留 **80GB-120GB**，不宜过小，否则系统运行时可能出现空间不足的问题
:::

## 第二步:安装 Linux Mint

### 1. 制作 Ventoy 启动盘

Ventoy 需要安装到一个 **单独的 U 盘** 上,用于引导 Linux Mint 安装

1. 解压 Ventoy 压缩包,双击运行 `Ventoy2Disk.exe`
2. 在 **"设备"** 下拉菜单中选择你的 **U 盘**(不是移动硬盘)
3. 分区类型选择 **GPT**
4. 点击 **"安装"**,确认两次提示后开始安装

::: danger 警告
请仔细确认设备名称和容量，**选错磁盘会导致该磁盘所有数据丢失**
:::

### 2. 复制 ISO 到 U 盘

将 Linux Mint ISO 文件直接复制到 Ventoy U 盘的根目录：

```
Ventoy U 盘/
├── linuxmint-22.1-cinnamon-64bit.iso
```

::: info 提示
Ventoy 的优势在于无需使用任何写入工具，像复制普通文件一样将 ISO 拖入 U 盘即可，启动时 Ventoy 会列出所有可用的 ISO 供你选择
:::

### 3. 从 Ventoy U 盘启动

1. 将 Ventoy U 盘和移动硬盘同时插入电脑
2. 重启并从 Ventoy U 盘启动
3. 在 Ventoy 菜单中选择 **linuxmint-22.1-cinnamon-64bit.iso** 启动
4. 选择 **"Start Linux Mint"** 进入 Live 桌面,Linux Mint 默认已经配置好中文环境

点击右下角网络图标,连接可用的 Wi-Fi 网络

### 4. 启动安装程序

在桌面双击 **"Install Linux Mint"** 图标

### 5. 安装设置

按照安装向导依次设置：

1. **语言**：选择 **"中文(简体)"**
2. **键盘布局**：选择 **"English (US)"**
3. **多媒体编解码器**：勾选 **"安装多媒体编解码器"** (推荐)
4. **安装类型**：选择 **"其他选项"** (手动分区)

::: danger 警告
**不要**选择"清除磁盘并安装"或"与 Windows 共存"，必须选择 **"其他选项"** 手动指定分区，否则会破坏已有的 WinToGo 分区
:::

### 6. 手动创建 Linux 分区

在分区设置界面中，找到 WinToGo 留下的 **"空闲"** 空间（约 410GB）：

1. 选中 **"空闲"** 空间,点击 **"+"** 按钮
2. 设置分区参数：
   - **大小**：`51200` MB (约 50GB)
   - **类型**：主分区
   - **文件系统**：`ext4`
   - **挂载点**：`/`
3. 点击 **"确定"**

![1.jpg](https://img.zhaizz.top/file/blog-website/win-linux-togo/iZHS5x8x.jpg)

::: info 提示
剩余的空闲空间暂时不要动，后续会用 DiskGenius 在 Windows 中创建存储分区
:::

### 7. 安装引导器位置

在安装界面底部，将 **"安装启动引导器的设备"** 选择为移动硬盘的 EFI 分区（通常是 `/dev/sda2`，带有 `Windows Boot Manager`）

::: danger 警告
引导器位置 **必须** 选择移动硬盘的 EFI 分区，不能选择本地硬盘，否则会覆盖本地系统的引导
:::

### 8. 完成安装

1. 设置时区
2. 设置用户名和密码
3. 点击 **"安装"**,等待安装完成
4. 安装完成后,重启并拔掉 Ventoy U 盘

### 9. 验证双系统启动

重启后从移动硬盘启动,应该会看到 GRUB 引导菜单:

| 选项 | 说明 |
| ---- | ---- |
| **Linux Mint 22.1 Cinnamon** | 启动 Linux Mint |
| **Windows Boot Manager** | 启动 Windows |

::: info 提示
如果 GRUB 菜单中没有显示 Windows 选项，进入 Linux Mint 后执行 `sudo update-grub` 即可重新扫描并添加 Windows 引导项
:::

## 第三步:创建存储分区

Linux 安装完成后，剩余的空闲空间需要创建为存储分区。由于 DiskGenius 是 Windows 工具，需要从 WinToGo 启动来操作

### 1. 从 WinToGo 启动

重启电脑,在 GRUB 菜单中选择 **Windows Boot Manager** 进入 Windows 系统

### 2. 运行 DiskGenius

从 [DiskGenius 官网](https://www.diskgenius.cn/download.php) 下载免费版并安装，打开 DiskGenius，在左侧磁盘列表中选择你的移动硬盘（通过容量大小来辨别）

::: danger 警告
请仔细确认选中的是移动硬盘，**不要误操作本地硬盘的分区**
:::

### 3. 确认当前分区状态

此时移动硬盘应该有以下分区：

| 分区 | 大小 | 文件系统 | 说明 |
| ---- | ---- | -------- | ---- |
| ESP | 550MB | FAT32 | EFI 引导分区 |
| Windows (C:) | 约 100GB | NTFS | Windows 系统分区 |
| linux | 50GB | ext4 | Linux Mint 系统分区 |
| 未分配 | 约 328GB | - | 剩余空间 |

### 4. 创建存储分区

1. 右键点击 **"未分配"** 空间,选择 **"建立新分区"**
2. 设置分区参数：
   - **文件系统类型**：选择 **`exFAT`** (推荐)
   - **分区大小**：使用全部剩余空间
   - **卷标**：`Data`
3. 点击 **"确定"**

::: info 提示
存储分区使用 **exFAT** 是因为 Windows 和 Linux 都能原生读写，且没有 FAT32 单文件 4GB 的限制
:::

### 5. 保存分区操作

确认分区布局无误后，点击左上角 **"保存更改"** 按钮，DiskGenius 会弹出确认对话框，确认后开始执行分区操作

最终的分区布局:

| 分区 | 大小 | 文件系统 | 用途 |
| ---- | ---- | -------- | ---- |
| ESP | 550MB | FAT32 | 引导分区 |
| Windows | 约 100GB | NTFS | Windows 系统 |
| Linux | 50GB | ext4 | Linux Mint 系统 |
| Data | 约 328GB | exFAT | 共享存储 |

## 第四步:挂载存储分区

存储分区创建完成后，需要重启进入 Linux Mint 配置自动挂载

### 1. 重启进入 Linux Mint

重启电脑,在 GRUB 菜单中选择 **Linux Mint** 进入系统

### 2. 查看分区信息

```bash
lsblk
```

找到存储分区（通常是最大的 exFAT 分区，例如 `/dev/sdb4`）

### 3. 安装文件系统支持

```bash
# exFAT 支持
sudo apt install -y exfatprogs

# NTFS 支持(用于读写 Windows 分区)
sudo apt install -y ntfs-3g
```

### 4. 创建挂载点并配置自动挂载

```bash
# 创建挂载点
sudo mkdir -p /mnt/data

# 获取存储分区的 UUID
sudo blkid /dev/sdb4
```

记下输出的 UUID 值，然后添加 fstab 配置：

```bash
# 备份 fstab
sudo cp /etc/fstab /etc/fstab.bak

# 添加自动挂载(将 UUID 替换为实际值)
echo 'UUID=XXXX-XXXX  /mnt/data  exfat  defaults,uid=1000,gid=1000,umask=000  0  0' | sudo tee -a /etc/fstab
```

### 5. 测试挂载

```bash
# 测试 fstab 配置
sudo mount -a

# 检查挂载结果
df -h | grep data
```

挂载成功后 `/mnt/data` 目录即可正常读写

## 常用软件安装

进入 Linux Mint 后，建议安装以下常用软件：

```bash
# 更新软件源
sudo apt update && sudo apt upgrade -y

# 安装常用工具
sudo apt install -y wget curl git vim net-tools

# 安装中文输入法(Fcitx5 + 拼音)
sudo apt install -y fcitx5 fcitx5-chinese-addons fcitx5-config-qt

# 安装 Chrome 浏览器
wget -q -O - https://dl.google.com/linux/linux_signing_key.pub | sudo gpg --dearmor -o /usr/share/keyrings/google-chrome.gpg
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/google-chrome.gpg] http://dl.google.com/linux/chrome/deb/ stable main" | sudo tee /etc/apt/sources.list.d/google-chrome.list
sudo apt update && sudo apt install -y google-chrome-stable

# 安装多媒体播放器
sudo apt install -y vlc ffmpeg
```

## 跨系统文件访问

### 在 Linux 中访问 Windows 分区

安装 NTFS 支持后即可读写 Windows 分区:

```bash
sudo apt install -y ntfs-3g

# 手动挂载 Windows 分区
sudo mkdir -p /mnt/windows
sudo mount -t ntfs-3g /dev/sdb2 /mnt/windows
```

如需开机自动挂载，在 `/etc/fstab` 中添加：

```
UUID=XXXX-XXXX  /mnt/windows  ntfs-3g  defaults,uid=1000,gid=1000,umask=000  0  0
```

### 在 Windows 中访问 Linux 分区

Windows 默认无法读取 ext4 格式的 Linux 分区，可以使用以下工具：

- [Ext2Fsd](http://www.ext2fsd.com/):开源的 Windows ext2/ext3/ext4 驱动
- [DiskGenius](https://www.diskgenius.cn/):国产磁盘工具,支持读取 ext4 分区

::: info 提示
日常使用中，建议将共享文件统一存放在存储分区中，避免频繁跨系统读写各自的系统分区
:::

## 注意事项

✅ 制作前务必备份移动硬盘上的重要数据

✅ 推荐使用 USB 3.0 及以上接口的移动固态硬盘,容量不低于 128GB

✅ 安装 Linux 时必须选择 **"其他选项"** 手动分区

✅ 安装引导器时**务必选择**移动硬盘的 EFI 分区,不要选到本地硬盘

✅ 共享文件建议统一存放在存储分区,方便两个系统互相访问

⚠️ 不要在系统运行时直接拔出移动硬盘，可能导致数据损坏或文件系统错误

⚠️ 部分电脑的安全启动 (Secure Boot) 可能会阻止移动硬盘启动，如遇问题请在 BIOS 中暂时关闭 Secure Boot

⚠️ Linux Mint 在不同电脑上可能遇到显卡驱动兼容问题，可尝试使用 compatibility mode 模式启动

⚠️ WinToGo 不支持 Windows 更新中的大版本升级，建议全新制作
