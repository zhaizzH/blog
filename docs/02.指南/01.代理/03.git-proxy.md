# Git Bash 使用代理

## 设置代理

```bash
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
```

>Git中粘贴使用shift+insert

## 验证代理是否生效

```bash
curl -I https://www.google.com
```

## 配置 Git 全局代理

```bash
# HTTP 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# SOCKS5 代理（推荐）# SOCKS5 代理（推荐）
git config --global http.proxy socks5://127.0.0.1:7891
git config --global https.proxy socks5://127.0.0.1:7891

## 取消 Git 代理## 取消 Git 代理
git config --global --unset http.proxygit config --global --unset http.proxy
git config --global --unset https.proxygit config --global --unset https.proxy
```

::: info 提示
如果修改了 Clash 的端口设置,请相应调整上述命令中的端口号
|     端口类型      | 默认值 |
| ---------------  | :-----:|
| HTTP 代理端口     |  7890  |
| SOCKS5 代理端口   |  7891  |
| 混合代理端口      |   7890  |
| RESTful API 端口 |   9090  |
:::

## 针对特定域名配置代理

如果只想为 GitHub 配置代理:

```bash
# 只为 GitHub 配置代理
git config --global http.https://github.com.proxy socks5://127.0.0.1:7891
```

取消特定域名代理:

```bash
git config --global --unset http.https://github.com.proxy
```

## 持久化配置

如果希望每次打开 Git Bash 都自动设置代理,可以将代理命令添加到 `~/.bashrc` 文件中:

```bash
echo 'export HTTP_PROXY="http://127.0.0.1:7890"' >> ~/.bashrc
echo 'export HTTPS_PROXY="http://127.0.0.1:7890"' >> ~/.bashrc
source ~/.bashrc
```
## 注意事项

✅ 环境变量在**当前终端会话**中有效

✅ 关闭终端后,临时设置会失效

✅ Git 全局配置会持久保存

⚠️ 确保 Clash 正在运行且代理端口未被占用
