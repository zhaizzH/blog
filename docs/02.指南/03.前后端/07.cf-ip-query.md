# 用 Cloudflare Workers 搭建 IP 地址查询工具

一个基于 Cloudflare Workers 的 IP 地址归属地与网络信息查询工具,支持双数据源切换,无需服务器,零成本部署,[示例网站](https://ip.zhaizz.top/)

![cf-ip-query.png](https://img.zhaizz.top/file/blog-website/evwl5Dr1.png)

## 功能

- **IP 归属地查询** — 国家、省份、城市、区县
- **网络信息** — 运营商、ASN、IP 类型、时区
- **坐标定位** — 经纬度坐标,一键跳转地图
- **双数据源** — ip9.com.cn(国内)和 ip.sb(国际),自动根据访客所在地选择
- **响应式设计** — 全端适配,移动端友好

## 项目结构

项目只有三个核心文件：

```
ip-query/
├── worker.js      # Workers 脚本(含前端页面)
├── wrangler.toml  # Wrangler 配置
└── package.json   # 依赖管理
```

所有代码都在 `worker.js` 中,包括后端 API 和前端 UI,部署零配置

## 部署方式

### 前置要求

- [Cloudflare 账号](https://dash.cloudflare.com)
- [Node.js](https://nodejs.org/) 16+

### 方式一：Wrangler CLI

```bash
# 安装 wrangler
npm install -g wrangler

# 登录 Cloudflare
wrangler login

# 部署
git clone https://github.com/<你的用户名>/ip-query
cd ip-query
wrangler deploy
```

### 方式二：GitHub 集成(推荐)

无需本地安装工具,Cloudflare Dashboard 直连 GitHub 仓库自动部署:

1. **推送代码到 GitHub**

   ```bash
   git init
   git add .
   git commit -m "init: IP 地址查询工具"
   git remote add origin https://github.com/<用户名>/<仓库名>.git
   git push -u origin main
   ```

2. **进入 Cloudflare Workers Dashboard**
   - 打开 https://dash.cloudflare.com/ → Workers 和 Pages
   - 点击 **创建** → **Worker** → **部署** 标签页

3. **连接 GitHub 仓库**
   - 点击 **Continue with GitHub**
   - 授权并选择刚才推送的仓库
   - 分支选择 `main`

4. **保存并部署**
   - 框架预设选择**无**
   - 其他保持默认,点击 **保存并部署**
   - 以后每次推送 `main` 分支都会自动重新部署

部署完成后,访问 `https://ip-query.<你的子域名>.workers.dev` 即可使用

### 绑定自定义域名(可选)

进入 Workers Dashboard → 你的 Worker → **触发器** → **自定义域名** → 添加域名,如 `ip.yourdomain.com`

## API 使用

### 查询 IP 信息

```
GET /api/query?ip=8.8.8.8&source=ipsb
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `ip` | string | 要查询的 IP(可选,留空查访客自身 IP) |
| `source` | string | 数据源：`auto`(默认)、`ip9`、`ipsb` |

### 响应格式

```json
{
  "success": true,
  "data": {
    "ip": "8.8.8.8",
    "country": "United States",
    "countryCode": "US",
    "region": "California",
    "city": "Mountain View",
    "isp": "Google LLC",
    "org": "Google LLC",
    "asn": "AS15169",
    "lat": 37.4056,
    "lng": -122.0775
  },
  "source": "ip.sb"
}
```

### 命令行测试

```bash
curl "https://ip-query.xxx.workers.dev/api/query?ip=8.8.8.8"
curl "https://ip-query.xxx.workers.dev/api/query?ip=114.114.114.114&source=ip9"
```

## 双数据源机制

代码通过访客的 `CF-IPCountry` 请求头判断所在地,自动选择最优数据源：

```js
if (country === 'CN') {
  data = await queryIP9(targetIP);  // 国内用户使用 ip9.com.cn
} else {
  data = await queryIPsb(targetIP); // 海外用户使用 ip.sb
}
```

用户也可手动切换数据源,覆盖自动选择

## 技术要点

- 前端直接内嵌在 Worker 中,零额外请求
- 使用 `CF-Connecting-IP` 获取访客真实 IP(无需额外配置)
- 国内地图跳转高德,海外跳转 Google Maps
- 全接口支持 CORS,可作为 API 给其他应用调用
