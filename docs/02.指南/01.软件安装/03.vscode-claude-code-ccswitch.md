# VS Code + Claude Code + CC-Switch 配置教程

Claude Code 是 Anthropic 推出的 AI 编程命令行工具，直接在终端中理解代码库、执行命令、编辑文件。CC-Switch 是一款开源的桌面应用，可统一管理 Claude Code 等多个 AI 编程 CLI 的 API 配置，支持一键切换 50+ 国内/国际供应商，解决国内用户无法直连 Claude API 的问题。

本文介绍从零开始配置 VS Code + Claude Code + CC-Switch 的完整流程。

## 准备工作

安装前确保以下环境已就绪：

| 软件 | 说明 | 验证命令 |
|------|------|---------|
| **Node.js** ≥ 18 | 运行 Claude Code 必需 | `node --version` |
| **Git** | Claude Code 依赖 Git 分析代码 | `git --version` |

### Node.js 安装

从 [nodejs.org](https://nodejs.org/) 下载 LTS 版本安装包，按向导完成安装。安装后在终端验证：

```bash
node --version
npm --version
```

### Git 安装

从 [git-scm.com](https://git-scm.com/) 下载对应系统安装包。

**Windows** 用户在安装时建议选择：
- 默认编辑器选 VS Code
- 调整 PATH 环境选 **Git from the command line and also from 3rd-party software**
- 行结束符选 **Checkout as-is, commit Unix-style line endings**

安装后在终端验证：

```bash
git --version
```

## 安装 Claude Code CLI

### 方式一：官方安装脚本(推荐)

**macOS / Linux：**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows PowerShell(管理员身份)：**
```powershell
irm https://claude.ai/install.ps1 | iex
```

### 方式二：npm 全局安装

```bash
npm install -g @anthropic-ai/claude-code
```

### 验证安装

```bash
claude --version
```

### 绕过首次登录引导

Claude Code 首次运行会弹出 Anthropic 账号登录页面。编辑配置文件可直接跳过：

打开 `~/.claude.json`(Windows 路径为 `C:\Users\你的用户名\.claude.json`)，在末尾添加：

```json
"hasCompletedOnboarding": true
```

添加后文件内容类似：

```json
{
  // ... 其他配置 ...
  "hasCompletedOnboarding": true
}
```

保存后终端输入 `claude` 即可直接进入对话。

## 安装 CC-Switch

CC-Switch 是由 [farion1231](https://github.com/farion1231/cc-switch) 开发的开源跨平台桌面应用，可在 Claude Code、Codex、Gemini CLI 等工具间一键切换供应商，无需手动修改配置。

### 下载安装

| 系统 | 安装方式 |
|------|---------|
| **Windows** | 从 [GitHub Releases](https://github.com/farion1231/cc-switch/releases) 下载 `CC-Switch-*-Windows.msi` 安装包，或 `*-Portable.zip` 绿色版 |
| **macOS** | `brew tap farion1231/ccswitch && brew install --cask cc-switch` |
| **Linux** | 下载 `.deb` / `.rpm` / `.AppImage` |

Windows 系统要求 **Win10 x64** 或更高版本。

## 配置 AI 供应商

CC-Switch 内置了 50+ 供应商预设模板，配置后即可在 Claude Code 中使用。

### 添加供应商

1. 打开 CC-Switch 主界面
2. 点击右上角 **+** 按钮
3. 在「预设」下拉框中选择所需供应商
4. 填入 API Key(到对应平台申请)
5. 点击「添加」

### 常用供应商

| 平台 | API Key 申请地址 | 免费额度 |
|------|-----------------|---------|
| **DeepSeek** | https://platform.deepseek.com/ | 注册赠额度 |
| **阿里百炼(千问)** | https://www.aliyun.com/product/bailian | 新用户赠额度 |
| **智谱 GLM** | https://open.bigmodel.cn/ | 注册赠额度 |
| **SiliconFlow(硅基流动)** | https://cloud.siliconflow.cn/ | 注册赠额度 |
| **OpenRouter** | https://openrouter.ai/ | — |

### 自定义模型配置

部分供应商需要指定模型名称。在 CC-Switch 界面底部 JSON 配置区域添加：

```json
"ANTHROPIC_MODEL": "模型名称"
```

例如阿里百炼使用 Qwen 系列模型：

```json
"ANTHROPIC_MODEL": "qwen3.6-plus"
```

### 启用供应商

- **主界面**：点击供应商卡片的「Enable」按钮
- **系统托盘**：右键托盘图标直接选择供应商

切换后 **Claude Code 无需重启**(v2.0.69+ 支持热重载)，立即生效。

## VS Code 集成

### 安装 Vs Code

从[官网](https://code.visualstudio.com/)中下载

### 安装扩展

1. 打开 VS Code
2. 按 `Ctrl+Shift+X` 打开扩展面板
3. 搜索 **Claude Code for VS Code**
4. 认准 Anthropic 官方出品，点击安装

### 配置扩展

安装后左侧侧边栏会出现 Claude Code 图标。若弹出登录页面：

1. 点击扩展的齿轮图标 → **扩展设置**
2. 找到 **Disable Login prompt**，勾选启用
3. 重启 VS Code

### CC-Switch 配合设置

在 CC-Switch 中打开 **设置 → 通用**，开启 **「跳过 Claude Code 初次安装确认」**，确保 VS Code 中的 Claude Code 能正常识别 CC-Switch 的配置。

### 开始使用

1. 在 VS Code 中打开一个项目文件夹
2. 点击侧边栏 Claude Code 图标
3. 输入消息开始对话

可在对话中输入 `/model` 查看当前使用的模型。

## 验证配置

终端运行 `claude` 或在 VS Code Claude Code 面板中发送：

```
你好，请简单介绍一下自己
```

如果能正常回复，说明配置成功。

### 常用环境变量

| 变量 | 说明 |
|------|------|
| `ANTHROPIC_BASE_URL` | API 中转地址 |
| `ANTHROPIC_API_KEY` | API 密钥 |
| `CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING` | 设为 `1` 禁用 adaptive thinking |

## 进阶配置

### Skills 管理

CC-Switch 提供 Skills 面板，可浏览和安装社区 Skills，自动同步到 `~/.claude/skills/` 目录。

也可手动添加：

```
/plugin marketplace add <作者>/<仓库>
/plugin install <插件名>
```

### MCP 服务器配置

CC-Switch 提供统一的 MCP 管理面板，支持 stdio / HTTP / SSE 三种传输协议，一次配置同步到 Claude Code 等多个工具。

也可在项目根目录创建 `.mcp.json` 手动配置。

## 常见问题

**Q：400 thinking type 错误**
A：设置环境变量 `CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING=1`

**Q：401 Unauthorized**
A：检查 API Key 是否正确，确认账户有额度

**Q：claude 命令找不到**
A：检查 npm 全局安装路径是否在 PATH 中。Windows 用户需添加 `C:\Users\你的用户名\AppData\Roaming\npm` 到 PATH

**Q：PowerShell 禁止运行 npm 脚本**
A：新装的 Windows(尤其是 Win11 24H2)默认 PowerShell 执行策略为 `Restricted`，会阻止运行 `.ps1` 脚本，导致 `npm` / `node` 等命令报错。解决办法：

- **方法 1：永久修改执行策略(推荐)**
  以管理员身份打开 PowerShell，运行：
  ```powershell
  Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```
- **方法 2：仅当前会话临时绕过**
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
  ```

**Q：VS Code 插件仍提示登录**
A：确认已勾选 Disable Login prompt，重启 VS Code

**Q：切换供应商后不生效**
A：Claude Code 支持热重载，但如果更换了命令行工具(如 Codex、Gemini CLI)需重启终端,注意安装的 skill 和 plugin 需要重新安装
