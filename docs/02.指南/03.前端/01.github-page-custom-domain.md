# GitHub Pages 配置自定义域名

## 1. 在 NameSilo 购买域名

1. 访问 NameSilo 官网:https://www.namesilo.com
2. 注册并登录账号
3. 搜索想要的域名,选择未被注册的域名加入购物车
4. 结算付款,NameSilo **默认免费域名隐私保护**,无需额外设置

## 2. 前期准备

- 已搭建好 GitHub Pages 静态站
- 拥有 NameSilo 已注册域名
- 注册登录 Cloudflare 官网:https://dash.cloudflare.com
- 提前记住自己 GitHub 用户名:`xxx.github.io`

## 3. 仓库添加 CNAME 文件

根据你的 Pages 部署位置放置:
- 根目录部署:仓库根目录新建 `CNAME`
- `/docs` 部署:在 `docs` 文件夹新建 `CNAME`

文件**只写纯域名**,不要加 `http/https:www`:
```
example.com
```
提交推送到仓库即可

## 4. GitHub Pages 绑定自定义域名

1. 进入仓库 → `Settings` → `Pages`
2. 下滑找到 **Custom domain**,填入你的域名
3. 保存后勾选 **Enforce HTTPS** 强制 HTTPS
4. 等待 GitHub 检测 DNS 验证通过
> DNS检测未通过可以暂时忽略

## 5. 域名接入 Cloudflare

### 5.1 Cloudflare 添加站点
1. Cloudflare 左侧 **Domains（域名） - Overview （概览）** 点击 **Add domain（添加域名）**
2. 输入裸域名,如 `example.com`
3. 选择 **Free 免费版**
4. 自动扫描原有 DNS,直接跳过继续
5. 复制分配给你的两条 Cloudflare NS 地址,示例:
   ```
   rosalie.ns.cloudflare.com
   ruben.ns.cloudflare.com
   ```

### 5.2 NameSilo 修改 Nameserver
1. NameSilo 进入 `Quick settings`
2. 点击`NameServers`右侧的 Edit
3. **删掉全部默认 NS**
4. 选择自定义 NS,粘贴刚才复制的两条 Cloudflare NS
5. 保存

> 可能会出现报错:提示「更改名称服务器任务仍处于开放状态」,不要反复点保存,**等待 10～30 分钟解锁**后再修改即可,是 NameSilo 防频繁修改限制

### 5.3 检测 NS 是否生效
1. 在线检测:https://whatsmydns.net/#NS/你的域名

2. CMD 本地检测命令:
```CMD
nslookup -type=NS zhaizz.top 1.1.1.1
```
返回 Cloudflare NS 即代表修改成功

## 6. Cloudflare DNS 解析配置

### 6.1 根域名配置（裸域 @）
添加 **4 条 A 记录**:
- 类型:`A`
- 名称:`@`（保存后会自动识别为根域名）
- 代理:橙色云开启
- 地址依次填写:
```
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

### 6.2 www 子域名配置
添加 1 条 CNAME 记录:
- 类型:`CNAME`
- 名称:`www`
- 目标:`你的用户名.github.io`
- 代理:橙色云开启

> 后续只在 Cloudflare 管理 DNS,NameSilo 无需再做任何解析

## 7. Cloudflare 其他配置

### 7.1 SSL/TLS 设置
1. `SSL/TLS` → 概述:选择 **严格**
2. `SSL/TLS` → 边缘证书
   - 开启:**始终使用 HTTPS**
   - 开启:**自动 HTTPS 重写**

### 7.2 缓存加速设置
- `配置缓存`:
  - 缓存级别:标准
  - 浏览器缓存 TTL:1 天

## 8. 检测是否生效

1. 浏览器访问 `https://域名` 能正常打开、带安全锁
2. F12 查看响应头:
   - Server 显示 `cloudflare` 代表 CDN 加速成功
   - 仍显示 `github.com` 是**国内运营商 DNS 缓存未刷新**,等待 1～24 小时自动恢复

本地刷新 DNS 命令:
```CMD
ipconfig /flushdns
```

## 注意
- 部署在 Cloudflare Workers 上的 VitePress 项目在二级页面刷新会出现找不到网页的情况,解决方法是**禁用永久链接**
