# VitePress 配置 RSS 订阅支持

使用 [粥里有勺糖](https://sugarat.top/aboutme.html) 的 [配置教程](https://www.cnblogs.com/roseAT/p/17649853.html)

## 1. 安装 RSS 插件

在 VitePress 项目根目录执行命令,安装 vitepress-plugin-rss 插件:

``` 
npm install vitepress-plugin-rss -D
pnpm add vitepress-plugin-rss -D
yarn add vitepress-plugin-rss -D
```

## 2. 前期准备

- 已搭建好 VitePress 静态博客
- 解析完成的自定义域名
- Cloudflare 已接入自定义域名,参考 [GitHub Pages 自定义域名配置流程](./github-page-custom-domain)

## 3. 修改 VitePress 配置文件

打开 docs/.vitepress/config.ts,添加 RSS 插件配置:

```
import { defineConfig } from 'vitepress'
import { RssPlugin } from 'vitepress-plugin-rss'

export default defineConfig({
  // 其他原有配置（如 title、themeConfig 等）...

  // RSS 插件配置（详细设置可以看 RSSOptions）
  vite: {
    plugins: [
      RssPlugin({
        title: 'example',           // RSS 显示的博客名称
        baseUrl: 'https://zhaizz.top', // 自定义域名
        limit: 20,                    // RSS 显示最新文章数量（0=显示全部）
        // 过滤不需要进入 RSS 的文章
        filter: (post) => {
          if (post.filepath.includes('@pages/')) return false
          if (post.filepath.endsWith('目录.md')) return false
          return true
        }
      })
    ]
  },

  head: [
    ['link', {rel: 'alternate',type: 'application/rss+xml',title: 'RSS 订阅',href: '/feed.rss'}]
  ]
})
```

> 插件会自动在博客导航栏添加 RSS 图标,无需手动配置

## 4. 本地构建测试 RSS

配置完成后,执行构建命令,生成 RSS 文件:

```
npm run docs:build
```

:::info 提示
构建成功后,在路径 docs/.vitepress/dist/feed.rss 中可找到生成的 RSS 文件

用浏览器打开该文件,能看到 XML 格式内容,且包含文章标题、日期,即为生成成功

然后就可以提交 github 部署了
:::

## 5. Cloudflare 放行 RSS

由于 Cloudflare 安全规则会拦截 RSS 阅读器请求,需配置两条规则放行 /feed.rss,确保订阅正常

### 5.1 WAF 自定义规则

1. 进入 Cloudflare 控制台域名设置中 → 左侧 安全性 → 安全规则
2. 创建规则,点击 `自定义规则`
3. 规则名称:填写 `放行 RSS 订阅`
4. 匹配条件:
        
  - 字段:选择 `URI 路径`
  - 运算符:选择 `等于`
  - 值:填写 `/feed.rss`

5. 执行操作:选择 跳过,勾选下方所有 要跳过的 WAF 组件
6. 点击右下角 部署,并将该规则拖动至规则列表最上方

如下图所示
![01.vitepress-rss.png](https://img.zhaizz.top/file/blog-website/vitepress-rss/1778138650628_01.vitepress-rss.png)

### 5.2 页面规则

1. 进入 Cloudflare 控制台域名设置中 → 左侧 规则 → 页面规则
2. 点击 `创建页面规则`
3. URL 匹配:填写你的 RSS 完整地址,如 https://zhaizz.top/feed.rss 
4. 添加两项设置:
        
  - 缓存级别:选择 `绕过`
  - SSL:选择 `关闭`

5. 点击 保存和部署页面规则

如下图所示

## 6. RSS 生效检测与验证

1. 等待 Cloudflare 规则生效
2. 浏览器访问 https://zhaizz.top/feed.rss （或者你自定义的文件名）,能直接打开 XML 内容
