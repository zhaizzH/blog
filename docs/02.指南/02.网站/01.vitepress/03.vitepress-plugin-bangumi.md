# VitePress Bangumi 插件使用指南

`vitepress-plugin-bangumi` 是一个为 VitePress 设计的 [Bangumi](https://bgm.tv) 收藏数据展示插件它能够自动从 Bangumi 平台拉取你的收藏数据,并通过卡片网格组件在博客中展示

## 功能特性

- **构建时自动拉取** — 在 VitePress 构建结束时,自动从 Bangumi API 获取收藏数据并输出为 JSON 文件
- **开发模式支持** — `vitepress dev` 模式下也会生成数据到 `public` 目录,并监听文件变化自动刷新
- **TypeScript 支持** — 提供完整的类型定义
- **内置展示组件** — 提供现成的 Vue 卡片网格组件,支持筛选、分页、进度显示

## 安装

```bash
npm install vitepress-plugin-bangumi
```

## 快速开始

### 1. 在 VitePress 配置中注册插件

在 `.vitepress/config.ts` 中引入并配置插件:

```typescript
import { defineConfig } from 'vitepress'
import { BangumiPlugin } from 'vitepress-plugin-bangumi'

export default defineConfig({
  vite: {
    plugins: [
      BangumiPlugin({
        userId: '你的Bangumi用户名',  // 必填
        subjectType: 2,               // 只拉取动画
        collectionType: [2, 3],       // 看过 + 在看
      }),
    ],
  },
})
```

### 2. 在主题中注册组件

在 `.vitepress/theme/index.ts` 中注册 `BangumiCollection` 组件:

```typescript
import BangumiCollection from 'vitepress-plugin-bangumi/client'

export default {
  enhanceApp({ app }) {
    app.component('BangumiCollection', BangumiCollection)
  }
}
```

### 3. 创建展示页面

创建页面文件 `docs/bangumi.md`（或 `docs/@pages/bangumiPage.md`）:

```markdown
---
title: 追番
permalink: /bangumi
layout: page
article: false
sidebar: false
---

<BangumiCollection />
```

然后在 `.vitepress/config.ts` 的 `themeConfig.nav` 中添加导航链接:

```typescript
nav: [
  { text: "首页", link: "/" },
  {
    text: "我的",
    items: [
      { text: "追番", link: "/bangumi" },
    ],
  },
]
```

### 4. 构建运行
使用 `npm run docs:build` 或 `npm run docs:dev`:

## 配置选项

### BangumiPlugin 选项

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `userId` | `string` | — | **必填**Bangumi 用户名 |
| `limit` | `number` | `50` | 每次请求的条目数上限（最大 100） |
| `outputFile` | `string` | `'bangumi.json'` | 输出的 JSON 文件名 |
| `subjectType` | `SubjectType \| SubjectType[]` | `[]` | 条目类型筛选,不传则获取所有类型 |
| `collectionType` | `CollectionType \| CollectionType[]` | `[]` | 收藏状态筛选,不传则获取所有状态 |
| `baseUrl` | `string` | `'https://api.bgm.tv/v0'` | Bangumi API 基础 URL |
| `userAgent` | `string` | `'vitepress-plugin-bangumi/{version}'` | 请求时使用的 User-Agent |

### 条目类型（SubjectType）

| 值 | 类型 | 颜色 |
|----|------|------|
| `1` | 书籍 | `#4a90d9` |
| `2` | 动画 | `#e74c3c` |
| `3` | 音乐 | `#27ae60` |
| `4` | 游戏 | `#f39c12` |
| `6` | 三次元 | `#8e44ad` |

### 收藏状态（CollectionType）

| 值 | 标签 | 说明 |
|----|------|------|
| `1` | 想看 | 计划观看 |
| `2` | 看过 | 已完成观看 |
| `3` | 在看 | 正在观看 |
| `4` | 搁置 | 暂时搁置 |
| `5` | 抛弃 | 已放弃 |

### BangumiCollection 组件 Props

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `data` | `BangumiSubject[]` | `[]` | 直接传入数据（跳过 fetch） |
| `src` | `string` | `'/bangumi.json'` | 数据 JSON 文件路径 |
| `pageSize` | `number` | `24` | 每页显示的条目数 |

## 配置示例

### 示例 1:只拉取动画

```typescript
BangumiPlugin({
  userId: 'your-id',
  subjectType: 2,  // 2 = 动画
})
```

### 示例 2:拉取多种类型

```typescript
BangumiPlugin({
  userId: 'your-id',
  subjectType: [2, 3, 4],  // 动画、音乐、游戏
})
```

### 示例 3:只拉取特定收藏状态

```typescript
BangumiPlugin({
  userId: 'your-id',
  collectionType: [2, 3],  // 只拉取"看过"和"在看"
})
```

### 示例 4:自定义输出文件名

```typescript
BangumiPlugin({
  userId: 'your-id',
  outputFile: 'my-collections.json',
})
```

然后在组件中指定路径:

```vue
<BangumiCollection src="/my-collections.json" />
```

### 示例 5:使用本地数据（跳过网络请求）

```vue
<script setup>
import data from '../public/bangumi.json'
import BangumiCollection from 'vitepress-plugin-bangumi/client'
</script>

<BangumiCollection :data="data" />
```

### 示例 6:自定义每页显示数量

```vue
<BangumiCollection :pageSize="20" />
```

## 工作流程

```
1. 执行构建命令（npm run docs:build 或 npm run docs:dev）
                    ↓
2. BangumiPlugin 触发,调用 Bangumi API
                    ↓
3. 自动分页拉取全部收藏数据
                    ↓
4. 数据写入文件
   - 构建模式:<outDir>/bangumi.json
   - 开发模式:public/bangumi.json
                    ↓
5. BangumiCollection 组件在浏览器中 fetch JSON
                    ↓
6. 渲染为卡片网格展示
```

## 进阶：客户端动态导入（SSR 兼容）

如果遇到 SSR 兼容性问题，可以创建一个包装器组件，使用 `<ClientOnly>` 和动态导入：

```vue
<!-- .vitepress/theme/components/BangumiCollectionWrapper.vue -->
<template>
  <ClientOnly>
    <component :is="Comp" v-bind="$attrs" />
  </ClientOnly>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue'

const Comp = ref()

onMounted(async () => {
  const mod = await import('vitepress-plugin-bangumi/client')
  Comp.value = mod.default
})
</script>
```

然后在 `.vitepress/theme/index.ts` 中注册包装器组件：

```typescript
import BangumiCollectionWrapper from './components/BangumiCollectionWrapper.vue'

export default {
  enhanceApp({ app }) {
    app.component('BangumiCollection', BangumiCollectionWrapper)
  },
}
```

## 常见问题

::: info 如何找到自己的 Bangumi 用户名？
访问 https://bgm.tv 并登录,你的用户名显示在个人资料页面的 URL 中例如 https://bgm.tv/user/zhaizz ,用户名就是 `zhaizz`
:::

::: info 开发时如何实时查看数据更新？
使用 `npm run docs:dev` 运行开发服务器,插件会自动检测 `.md` 文件变化并刷新数据数据会保存到 `public/bangumi.json`
:::

::: info 组件不显示内容？
1. 检查 Bangumi 用户名是否正确
2. 确保网络连接正常,能访问 Bangumi API
3. 检查浏览器控制台是否有错误信息
4. 验证数据文件 `public/bangumi.json` 是否存在且有有效内容
:::

::: info 如何自定义卡片样式？
在 `.vitepress/theme/styles/custom.css` 中添加 CSS 规则来覆盖组件样式组件使用的类名可以通过浏览器开发者工具查看
:::
