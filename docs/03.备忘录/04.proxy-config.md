# 代理配置

## 通用说明

> 默认情况下，Clash 的本地代理端口为 `7890`。如果修改了 Clash 的端口设置，请相应调整命令中的端口号。

::: info 提示
Clash 默认端口

| 端口类型 | 默认值 |
| --------------- | :-----:|
| HTTP 代理端口 | 7890 |
| SOCKS5 代理端口 | 7891 |
| 混合代理端口 | 7890 |
| RESTful API 端口 | 9090 |
:::

## Windows

### CMD

```cmd
set http_proxy=http://127.0.0.1:7890
set https_proxy=http://127.0.0.1:7890
```

验证代理是否生效：

```cmd
curl -I https://www.google.com
```

取消代理：

```cmd
set http_proxy=
set https_proxy=
```

> 这些环境变量仅在**当前 CMD 窗口**中有效,关闭窗口后自动失效

### PowerShell

```powershell
$Env:HTTP_PROXY = "http://127.0.0.1:7890"
$Env:HTTPS_PROXY = "http://127.0.0.1:7890"
```

验证代理是否生效：

```powershell
Invoke-WebRequest -Uri "https://www.google.com" -Method Head -UseBasicParsing
```

取消代理：

```powershell
$Env:HTTP_PROXY = $null
$Env:HTTPS_PROXY = $null
```

> 环境变量在**当前 PowerShell 会话**中有效,通过 `$PROFILE` 配置可实现持久化

## Linux / macOS

```bash
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
export ALL_PROXY="socks5://127.0.0.1:7891"
```

验证代理是否生效：

```bash
curl -I https://www.google.com
```

取消代理：

```bash
unset HTTP_PROXY
unset HTTPS_PROXY
unset ALL_PROXY
```

> 写入 `~/.bashrc` 或 `~/.zshrc` 可实现持久化

## Git

### 环境变量方式

```bash
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
```

### Git 全局代理配置

```bash
# HTTP 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# SOCKS5 代理（推荐）
git config --global http.proxy socks5://127.0.0.1:7891
git config --global https.proxy socks5://127.0.0.1:7891
```

### 针对特定域名配置代理

```bash
# 只为 GitHub 配置代理
git config --global http.https://github.com.proxy socks5://127.0.0.1:7891
```

取消特定域名代理：

```bash
git config --global --unset http.https://github.com.proxy
```

### 取消 Git 代理

```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

> Git 全局配置会持久保存,无需每次重新设置
