# Maven 分模块开发与聚合继承

## 分模块开发

### 分模块开发设计

将原始模块按照功能拆分成若干个子模块，方便模块间的相互调用和接口共享。

![分模块设计](https://img.zhaizz.top/file/blog-website/maven/1630768703430.png)

![分层拆分](https://img.zhaizz.top/file/blog-website/maven/1630768869208.png)

### 分模块开发实现

#### 抽取 domain 层

1. 创建新模块 `maven_03_pojo`（jar 项目）
2. 将 domain 类拷贝到新模块
3. 在原项目中删除 domain 包
4. 在原项目 pom.xml 添加新模块依赖

```xml
<dependency>
    <groupId>com.itheima</groupId>
    <artifactId>maven_03_pojo</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>
```

5. 使用 `mvn install` 将新模块安装到本地仓库

#### 抽取 Dao 层

1. 创建新模块 `maven_04_dao`（jar 项目）
2. 将 Dao 类拷贝到新模块，并添加相关依赖
3. 在原项目中删除 Dao 包并添加新模块依赖

#### 实现步骤总结

1. 创建 Maven 模块
2. 书写模块代码
3. 通过 `mvn install` 安装模块到本地仓库

## 聚合

将多个模块组织成一个整体，同时进行项目构建的过程称为聚合。

![聚合](https://img.zhaizz.top/file/blog-website/maven/1630858596147(1).png)

- 聚合工程通常是一个不具有业务功能的「空」工程（有且仅有一个 pom 文件）
- 使用聚合工程可以将多个工程编组，实现对所包含的模块进行同步构建

### 配置步骤

1. 创建空的 Maven 项目
2. 打包方式设为 pom

```xml
<packaging>pom</packaging>
```

3. 添加所要管理的项目

```xml
<modules>
    <module>../maven_02_ssm</module>
    <module>../maven_03_pojo</module>
    <module>../maven_04_dao</module>
</modules>
```

## 继承

子工程可以继承父工程中的配置信息，常见于依赖关系的继承。

![继承](https://img.zhaizz.top/file/blog-website/maven/1630860344968(1).png)

### 配置步骤

1. 在子项目中设置其父工程

```xml
<parent>
    <groupId>com.itheima</groupId>
    <artifactId>maven_01_parent</artifactId>
    <version>1.0-RELEASE</version>
    <relativePath>../maven_01_parent/pom.xml</relativePath>
</parent>
```

2. 将子项目共同使用的 jar 包抽取到父工程的 `pom.xml` 中
3. 删除子项目中已被抽取的重复依赖

### 依赖版本管理

使用 `dependencyManagement` 标签统一管理版本，子项目引入时无需指定 version：

```xml
<!-- 父工程 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

```xml
<!-- 子项目：无需指定 version -->
<dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <scope>test</scope>
</dependency>
```

## 聚合与继承的区别

| 对比项 | 聚合 | 继承 |
|--------|------|------|
| 作用 | 快速构建项目，管理项目 | 快速配置和管理 jar 包版本 |
| 配置位置 | 当前模块中配置关系 | 子模块中配置关系 |
| 感知关系 | 聚合可以感知到参与聚合的模块 | 父模块无法感知哪些子模块继承了自己 |

两者相同点：打包方式均为 pom，可以将两种关系制作到同一个 pom 文件中。
