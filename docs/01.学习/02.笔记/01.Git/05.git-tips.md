# Git 最佳实践与进阶技巧

## 最佳实践

### 提交信息格式

```text
<type>(<scope>): <short summary>
  │       │             │
  │       │             └─⫸ 紧凑简短的描述，无需大写，也不需要用句号结尾
  │       │
  │       └─⫸ Commit 范围: animations|common|core|router|docs...
  │
  └─⫸ Commit 类型
```

- `feat!` / `feat(scope)!` — Breaking Change（重构 API）
- `type(scope): subject` — 带范围的提交

### 类型

```text
feat:     新特性
fix:      修复 Bug
docs:     仅修改文档
style:    格式（空格、分号等，不影响代码逻辑）
refactor: 重构（非修复 Bug 或添加新特性）
perf:     性能优化
test:     添加或修改测试
build:    构建系统或外部依赖（gulp, npm）
ci:       CI 配置文件或脚本（GitHub Action, Travis）
chore:    辅助工具、库等（不影响源码或测试）
revert:   回退代码
```

### 分支命名

```text
main/master            # 主分支
develop                # 开发分支
feature/feature-name   # 功能分支
bugfix/bug-name        # 修复分支
hotfix/hotfix-name     # 热修复分支
release/v1.0.0         # 发布分支
```

### 工作流

```bash
# 功能开发
git checkout -b feature/new-feature develop
# 开发...
git add .
git commit -m "feat: add new feature"
git checkout develop
git merge --no-ff feature/new-feature
git branch -d feature/new-feature
git push origin develop
```

### 保持整洁

```bash
# 定期同步主分支
git checkout main
git pull
git checkout feature-branch
git rebase main

# 压缩提交
git rebase -i HEAD~3

# 清理分支
git branch -d old-branch
git push origin --delete old-branch
```

## 常用命令组合

### 快速提交

```bash
git add . && git commit -m "message" && git push
```

### 撤销推送

```bash
git reset --hard HEAD~1
git push --force
```

### 同步 fork

```bash
git remote add upstream url
git fetch upstream
git checkout main
git merge upstream/main
git push origin main
```

### 查找提交

```bash
git log --all --grep="keyword"
git log --all --oneline | grep "keyword"
```

### 统计信息

```bash
git shortlog -sn       # 贡献者统计
git log --author="name" --oneline | wc -l # 提交数
git diff --stat        # 变更统计
```

### 备份分支

```bash
git branch backup-branch
git tag backup-tag
```

## Git Hooks

### 常用钩子

```text
pre-commit             # 提交前
commit-msg             # 提交消息
pre-push               # 推送前
post-checkout          # 检出后
post-merge             # 合并后
```

### 创建钩子

```bash
# .git/hooks/pre-commit
#!/bin/sh
npm test
if [ $? -ne 0 ]; then
  echo "Tests failed"
  exit 1
fi
chmod +x .git/hooks/pre-commit
```

### Husky (现代 Git Hooks)

```bash
npm install husky -D
npx husky init
echo "npm test" > .husky/pre-commit
```

## 技巧与诀窍

### 临时忽略文件

```bash
git update-index --assume-unchanged file
git update-index --no-assume-unchanged file
git ls-files -v | grep "^h" # 查看被忽略的文件
```

### 修改作者

```bash
git commit --amend --author="Name <email>"
git rebase -i HEAD~3 # 交互式修改多个
```

### 查看远程分支

```bash
git branch -r          # 远程分支
git branch -a          # 所有分支
git remote show origin # 远程仓库详情
```

### 优化仓库

```bash
git gc                 # 垃圾回收
git prune              # 清理无用对象
git fsck               # 检查完整性
git count-objects -vH  # 查看仓库大小
```

### 找回删除的提交

```bash
git reflog             # 查找提交hash
git checkout -b recover-branch commit-hash
git cherry-pick commit-hash
```

### 批量修改历史

```bash
# 修改邮箱
git filter-branch --env-filter '
OLD_EMAIL="old@email.com"
NEW_EMAIL="new@email.com"
if [ "$GIT_COMMITTER_EMAIL" = "$OLD_EMAIL" ]; then
  export GIT_COMMITTER_EMAIL="$NEW_EMAIL"
  export GIT_AUTHOR_EMAIL="$NEW_EMAIL"
fi' --tag-name-filter cat -- --branches --tags
```

## Patch 补丁

### 创建补丁

```bash
git diff commit_id > my.patch            # 两次提交差异生成补丁
git format-patch commit_id -n            # 指定 commit 之前的 n 次提交
git format-patch commit_id -1            # 指定单个提交生成补丁
git format-patch -3                      # 最近 3 次提交生成补丁
```

### 应用补丁

```bash
git apply --check path/to/my.patch       # 检查补丁能否正常应用
git apply < path/to/my.patch             # 应用补丁（不提交）
git apply --reject path/to/my.patch      # 自动合入不冲突部分，保留冲突
git am < email.txt                       # 应用 email 格式补丁（保留作者信息）
```

## 仓库迁移

### 迁移到新地址

```bash
git clone --bare https://old.com/user/project.git
cd project.git
git push --mirror https://new.com/user/newproject.git
git clone https://new.com/user/newproject.git
```

### Fork 仓库同步上游

```bash
git remote add upstream https://github.com/upstream/repo.git
git fetch upstream
git rebase upstream/main
git push -f
```

## 统计查询

### 个人代码量

```bash
git log --author="username" --pretty=tformat: --numstat | awk \
'{ add += $1; subs += $2; loc += $1 - $2 } END { printf "added: %s, removed: %s, total: %s\n", add, subs, loc }'
```

### 提交者排名

```bash
git log --pretty='%aN' | sort | uniq -c | sort -k1 -n -r | head -n 10
```

### 仓库体积

```bash
git count-objects -vH                    # 仓库总大小
git ls-files | xargs -r du -hs          # 所有文件大小
```

### 历史大文件

```bash
git rev-list --objects --all | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | awk '/^blob/ {print substr($0,6)}' | sort --numeric-sort --key=2 --reverse | head -n 10
```

### 提交数统计

```bash
git log --oneline | wc -l               # 总提交数
git log --since='2 weeks ago' --oneline  # 最近两周提交
```

## 进阶技巧

### 浅克隆与稀疏检出

```bash
git clone --depth=1 url                 # 仅克隆最新一次提交
git sparse-checkout init                # 初始化稀疏检出
git sparse-checkout set src/ tests/     # 仅检出指定目录
```

### 导出与导入分支

```bash
git bundle create backup.bundle branch  # 导出分支为文件
git clone backup.bundle repo -b branch  # 从文件导入分支
```

### 获取远程 PR

```bash
git fetch origin pull/123/head:pr-123   # 拉取远程 PR 到本地分支
```

### 重置第一个 commit

```bash
git update-ref -d HEAD                  # 清空所有 commit，改动回到工作区
```

### 创建空分支（无历史）

```bash
git checkout --orphan new-branch        # 新建无历史分支
git rm -rf .                            # 清空工作区
# 添加新文件...
git commit -m "Initial commit"
```

### 按提交日期排序分支

```bash
git for-each-ref --sort=-committerdate --format='%(refname:short)' refs/heads
```

### 查看被忽略/未跟踪的文件

```bash
git ls-files --others                   # 未跟踪文件
git ls-files --others -i --exclude-standard # 被忽略的文件
git ls-files -t                         # 所有 tracked 文件及其状态
```

### 详细行级差异

```bash
git diff --word-diff                    # 按单词显示差异
```

### 查看最近 tag

```bash
git describe --tags --abbrev=0          # 最近的 tag 名称
```

### 清除 gitignore 记录的文件

```bash
git clean -X -f                         # 删除被忽略的文件
git status --ignored                    # 显示忽略的文件
```
