# Git 暂存与高级操作

## 暂存 (Stash)

### stash 操作

```bash
git stash              # 暂存当前修改
git stash save "message" # 暂存并添加说明
git stash -u           # 包含未跟踪文件
git stash list         # 查看暂存列表
git stash show         # 查看最新暂存
```

### 恢复暂存

```bash
git stash pop          # 恢复最新暂存并删除
git stash apply        # 恢复最新暂存（保留）
git stash apply stash@{2} # 恢复指定暂存
git stash drop         # 删除最新暂存
git stash clear        # 清空所有暂存
```

### 从暂存创建分支

```bash
git stash branch name  # 从暂存创建分支
```

## 高级操作

### cherry-pick 挑选提交

```bash
git cherry-pick commit-hash    # 应用指定提交
git cherry-pick commit1 commit2 # 应用多个提交
git cherry-pick --continue     # 继续
git cherry-pick --abort        # 取消
```

### reflog 引用日志

```bash
git reflog             # 查看引用历史
git reflog show branch # 查看分支历史
git reset --hard HEAD@{2} # 恢复到之前的状态
```

### bisect 二分查找问题

```bash
git bisect start       # 开始二分查找
git bisect bad         # 标记当前为坏的
git bisect good commit # 标记好的提交
git bisect reset       # 结束查找
```

### clean 清理未跟踪文件

```bash
git clean -n           # 预览要删除的文件
git clean -f           # 删除未跟踪文件
git clean -fd          # 删除文件和目录
git clean -fX          # 删除被忽略的文件
git clean -fx          # 删除所有未跟踪文件
```

### worktree 工作树

```bash
git worktree add ../path branch # 添加工作树
git worktree list      # 列出工作树
git worktree remove path # 删除工作树
git worktree prune     # 清理工作树
```

## 子模块

### 添加子模块

```bash
git submodule add url path     # 添加子模块
git submodule init     # 初始化子模块
git submodule update   # 更新子模块
git submodule update --init --recursive # 递归初始化
```

### 管理子模块

```bash
git submodule status   # 查看子模块状态
git submodule foreach git pull # 更新所有子模块
git clone --recurse-submodules url # 克隆包含子模块
```

### 删除子模块

```bash
git submodule deinit path      # 注销子模块
git rm path            # 删除子模块
rm -rf .git/modules/path # 清理.git目录
```

## .gitignore

### 忽略规则

```txt
# 注释
*.log              # 忽略所有.log文件
!important.log     # 不忽略
/temp              # 仅忽略根目录的temp
temp/              # 忽略所有temp目录
**/logs            # 忽略所有logs目录
*.txt              # 忽略所有txt文件
```

### 常见模式

```txt
# Node.js
node_modules/
npm-debug.log

# Python
__pycache__/
*.py[cod]
.env

# IDE
.vscode/
.idea/
*.swp
```

### 全局忽略

```bash
git config --global core.excludesfile ~/.gitignore_global
```

### 清除已跟踪文件

```bash
git rm -r --cached .   # 清除缓存
git add .              # 重新添加
git commit -m "Update .gitignore"
```

## .gitattributes

### 行尾处理

```text
# 默认行为
* text=auto

# 始终规范化为 lf
*.c text
*.h text

# 检出时始终使用 CRLF
*.sln text eol=crlf

# 二进制文件不修改
*.png binary
*.jpg binary
```

### 语言统计 (linguist)

```text
# 标记生成文件，忽略语言统计
search/index.json linguist-generated=true

# 统计 SQL 文件
*.sql linguist-detectable=true

# 从统计中排除
docs/formatter.rb linguist-documentation=false
special-vendored-path/* linguist-vendored

# 将 .rb 文件检测为 Java
*.rb linguist-language=Java
```

## 冲突解决

### 查看冲突

```bash
git status             # 查看冲突文件
git diff               # 查看冲突内容
git log --merge        # 查看冲突的提交
```

### 解决冲突

```text
<<<<<<< HEAD
当前分支的内容
=======
合并分支的内容
>>>>>>> branch
```

标记为已解决：

```bash
git add conflicted-file
git commit
```

### 冲突工具

```bash
git mergetool          # 使用合并工具
git checkout --ours file   # 使用当前分支版本
git checkout --theirs file # 使用合并分支版本
```

### 取消合并

```bash
git merge --abort      # 取消合并
git rebase --abort     # 取消变基
```
