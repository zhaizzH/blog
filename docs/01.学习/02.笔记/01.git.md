# Git 笔记

## 配置

### 配置用户信息

```bash
git config --global user.name "Your Name"
git config --global user.email "email@example.com"
git config --list          # 查看所有配置
git config user.name       # 查看特定配置
```

### 配置级别

```bash
git config --system   # 系统级（所有用户）
git config --global   # 用户级（当前用户）
git config --local    # 仓库级（当前仓库，默认）
```

### 配置别名

```bash
git config --global alias.s status
git config --global alias.co checkout
git config --global alias.b branch
git config --global alias.cm commit
```

### 实用配置

```bash
# 自动换行（提交时转为 lf，检出时转为本机格式）
git config --global core.autocrlf input

# 中文文件名显示为实际名称（解决乱码）
git config --global core.quotepath false

# 设置大小写敏感
git config core.ignorecase false

# 忽略文件权限变化
git config core.fileMode false
```

### 代理配置

```bash
# 查看代理
git config --global http.proxy
git config --global https.proxy

# 设置 HTTP/SOCKS 代理（全局）
git config --global http.proxy http://127.0.0.1:1080
git config --global https.proxy http://127.0.0.1:1080

# 仅对指定域名设置代理
git config --global http.https://github.com.proxy socks5://127.0.0.1:1080

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
git config --global --unset http.https://github.com.proxy
```

```bash
# SSH 协议代理（编辑 ~/.ssh/config）
Host github.com
  ProxyCommand nc -X 5 -x 127.0.0.1:1080 %h %p

Host gitlab.com
  ProxyCommand nc -X 5 -x 127.0.0.1:1080 %h %p
```

## 初始化

### 创建仓库

```bash
git init               # 初始化本地仓库
git init project       # 创建并初始化项目目录
git clone url          # 克隆远程仓库
git clone url dir      # 克隆到指定目录
```

### 克隆选项

```bash
git clone --depth 1 url        # 浅克隆（仅最新提交）
git clone --branch dev url     # 克隆指定分支
git clone --single-branch url  # 仅克隆一个分支
```

## 基本操作

### 查看状态

```bash
git status             # 查看状态
git status -s          # 简短输出
git status -b          # 显示分支信息
git diff               # 查看未暂存的修改
git diff --staged      # 查看已暂存的修改
```

### 添加文件

```bash
git add file.txt       # 添加单个文件
git add .              # 添加所有文件
git add *.js           # 添加所有js文件
git add -p             # 交互式添加（部分添加）
```

### 提交

```bash
git commit -m "message"        # 提交
git commit -am "message"       # 添加并提交（已跟踪文件）
git commit --amend             # 修改最近一次提交
git commit --amend --no-edit   # 修改提交但不改消息
```

### 撤销操作

```bash
git restore file.txt   # 撤销工作区修改
git restore --staged file.txt  # 取消暂存
git reset HEAD~1       # 撤销最近一次提交（保留修改）
git reset --hard HEAD~1 # 撤销提交并丢弃修改
git revert commit-hash  # 创建新提交来撤销
```

### 删除文件

```bash
git rm file.txt        # 删除文件并暂存
git rm --cached file.txt # 仅从仓库删除（保留本地）
git mv old.txt new.txt # 重命名文件
```

## 分支管理

### 分支操作

```bash
git branch             # 查看本地分支
git branch -a          # 查看所有分支
git branch name        # 创建分支
git branch -d name     # 删除分支
git branch -D name     # 强制删除分支
git branch -m old new  # 重命名分支
```

### 切换分支

```bash
git checkout branch    # 切换分支
git checkout -b branch # 创建并切换分支
git switch branch      # 切换分支（新）
git switch -c branch   # 创建并切换（新）
```

### 合并分支

```bash
git merge branch       # 合并分支
git merge --no-ff branch # 禁用快速合并
git merge --squash branch # 压缩合并
git merge --abort      # 取消合并
```

### 变基

```bash
git rebase branch      # 变基到指定分支
git rebase -i HEAD~3   # 交互式变基最近3次提交
git rebase --continue  # 继续变基
git rebase --abort     # 取消变基
```

### 分支比较

```bash
git diff branch1 branch2       # 比较分支
git diff branch1..branch2      # 同上
git diff branch1...branch2     # 比较分支分叉点后的差异
git log branch1..branch2       # 查看差异提交
```

## 远程操作

### 远程仓库

```bash
git remote             # 查看远程仓库
git remote -v          # 查看远程仓库详细信息
git remote add origin url # 添加远程仓库
git remote remove origin  # 删除远程仓库
git remote rename old new # 重命名远程仓库
```

### 获取更新

```bash
git fetch              # 获取远程更新
git fetch origin       # 获取指定远程仓库
git fetch --all        # 获取所有远程仓库
git fetch --prune      # 获取并清理已删除的远程分支
```

### 拉取

```bash
git pull               # 拉取并合并
git pull --rebase      # 拉取并变基
git pull origin main   # 从指定远程分支拉取
git pull --ff-only     # 仅快速合并
```

### 推送

```bash
git push               # 推送到远程
git push origin main   # 推送到指定分支
git push -u origin main # 推送并设置上游
git push --all         # 推送所有分支
git push --tags        # 推送所有标签
git push --force       # 强制推送（慎用）
```

### 跟踪分支

```bash
git branch -vv         # 查看跟踪分支
git branch -u origin/main # 设置上游分支
git push -u origin branch # 推送并设置上游
git checkout --track origin/branch # 创建跟踪分支
```

```bash

git log --all          # 显示所有分支
```


### 查看提交

```bash

```

### 查看差异

```bash

git diff HEAD          # 工作区 vs 最新提交
git diff --cached      # 暂存区 vs 最新提交
git diff commit1 commit2 # 比较两次提交
git diff branch1 branch2 # 比较分支
```

### 查看文件历史

```bash
git log -- file.txt    # 文件提交历史
git log -p file.txt    # 显示文件变更内容
git blame file.txt     # 查看每行最后修改
git log --follow file.txt # 跟踪重命名
```

## 暂存 (Stash)

### stash 操作

```bash
git stash              # 暂存当前修改
git stash save "message" # 暂存并添加说明
git stash -u           # 包含未跟踪文件
git stash list         # 查看暂存列表
git stash show         # 查看最新暂存
```

### 恢复暂存

```bash
git stash pop          # 恢复最新暂存并删除
git stash apply        # 恢复最新暂存（保留）
git stash apply stash@{2} # 恢复指定暂存
git stash drop         # 删除最新暂存
git stash clear        # 清空所有暂存
```

### 从暂存创建分支

```bash
git stash branch name  # 从暂存创建分支
```

## 高级操作

### cherry-pick 挑选提交

```bash
git cherry-pick commit-hash    # 应用指定提交
git cherry-pick commit1 commit2 # 应用多个提交
git cherry-pick --continue     # 继续
git cherry-pick --abort        # 取消
```

## 最佳实践

### 提交信息格式

```text
<type>(<scope>): <short summary>
  │       │             │
  │       │             └─⫸ 紧凑简短的描述，无需大写，也不需要用句号结尾
  │       │
  │       └─⫸ Commit 范围: animations|common|core|router|docs...
  │
  └─⫸ Commit 类型
```

- `feat!` / `feat(scope)!` — Breaking Change（重构 API）
- `type(scope): subject` — 带范围的提交

### 类型

```text
feat:     新特性
fix:      修复 Bug
docs:     仅修改文档
style:    格式（空格、分号等，不影响代码逻辑）
refactor: 重构（非修复 Bug 或添加新特性）
perf:     性能优化
test:     添加或修改测试
build:    构建系统或外部依赖（gulp, npm）
ci:       CI 配置文件或脚本（GitHub Action, Travis）
chore:    辅助工具、库等（不影响源码或测试）
revert:   回退代码
```

### 分支命名

```text
main/master            # 主分支
develop                # 开发分支
feature/feature-name   # 功能分支
bugfix/bug-name        # 修复分支
hotfix/hotfix-name     # 热修复分支
release/v1.0.0         # 发布分支
```

## Patch 补丁

### 创建补丁

```bash
git diff commit_id > my.patch            # 两次提交差异生成补丁
git format-patch commit_id -n            # 指定 commit 之前的 n 次提交
git format-patch commit_id -1            # 指定单个提交生成补丁
git format-patch -3                      # 最近 3 次提交生成补丁
```

### 应用补丁

```bash
git apply --check path/to/my.patch       # 检查补丁能否正常应用
git apply < path/to/my.patch             # 应用补丁（不提交）
git apply --reject path/to/my.patch      # 自动合入不冲突部分，保留冲突
git am < email.txt                       # 应用 email 格式补丁（保留作者信息）
```


### 查看最近 tag

```bash
git describe --tags --abbrev=0          # 最近的 tag 名称
```

### 清除 gitignore 记录的文件

```bash
git clean -X -f                         # 删除被忽略的文件
git status --ignored                    # 显示忽略的文件
```
