# Docker Compose

Docker Compose 是一个用于定义和运行多容器 Docker 应用的工具。通过一个 YAML 文件来配置应用的服务，然后使用一个命令创建并启动所有服务。

> Docker Compose V2 是 Docker 官方推出的新版本，以 Docker 插件形式安装，命令从 `docker-compose`（带连字符）改为 `docker compose`（无连字符）。

## 安装

```bash
# 创建 root 用户的插件目录
mkdir -p /root/.docker/cli-plugins

# 下到二进制文件
curl -SL https://v4.gh-proxy.org/https://github.com/docker/compose/releases/download/v2.29.2/docker-compose-linux-x86_64 \
  -o /root/.docker/cli-plugins/docker-compose

# 加执行权限
chmod +x /root/.docker/cli-plugins/docker-compose

# 验证
docker compose version
```

## 基本命令

| 命令 | 说明 |
|------|------|
| `docker compose up -d` | 后台启动所有服务 |
| `docker compose down` | 停止并移除容器、网络 |
| `docker compose ps` | 列出项目中的容器 |
| `docker compose logs` | 查看服务日志 |
| `docker compose build` | 构建或重新构建服务 |
| `docker compose exec <服务名> bash` | 进入运行中的容器 |
| `docker compose restart` | 重启所有服务 |
| `docker compose stop` | 停止所有服务 |
| `docker compose start` | 启动已停止的服务 |
| `docker compose rm` | 删除已停止的容器 |
| `docker compose pull` | 拉取服务依赖的镜像 |
| `docker compose config` | 验证并查看 compose 文件配置 |

## compose.yml 基础结构

```yaml
services:
  web:
    build: .                    # 使用当前目录的 Dockerfile 构建
    ports:
      - "80:80"                 # 端口映射
    volumes:
      - ./code:/app             # 挂载目录
    depends_on:
      - db                      # 依赖的其他服务
    environment:
      - DB_HOST=db              # 环境变量
    networks:
      - frontend                # 连接的网络

  db:
    image: mysql:8.0            # 使用镜像
    volumes:
      - db_data:/var/lib/mysql  # 数据卷
    environment:
      MYSQL_ROOT_PASSWORD: password
    networks:
      - frontend

networks:
  frontend:                     # 定义网络

volumes:
  db_data:                      # 定义数据卷
```

## 常用配置说明

### 构建配置

```yaml
services:
  app:
    build:
      context: ./app            # Dockerfile 所在目录
      dockerfile: Dockerfile    # Dockerfile 文件名
      args:                     # 构建参数
        - APP_VERSION=1.0
      cache_from:
        - myimage:latest        # 缓存来源
```

### 环境变量

```yaml
services:
  app:
    environment:
      - DB_HOST=localhost       # 方式一：列表形式
      - DB_PORT=3306
    env_file:                   # 方式二：从文件读取
      - .env
      - ./config/app.env
```

### 端口映射

```yaml
services:
  web:
    ports:
      - "80:80"                 # host:container
      - "8080:8080"
      - "127.0.0.1:8081:8081"  # 绑定特定 IP
```

### 数据卷

```yaml
services:
  db:
    volumes:
      - db_data:/var/lib/mysql           # 命名卷
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql  # 绑定挂载
      - /var/log:/logs:ro                # 只读挂载

volumes:
  db_data:                        # 声明命名卷
    driver: local
```

### 网络配置

```yaml
services:
  web:
    networks:
      - frontend
      - backend
    network_mode: "host"         # 使用宿主机网络（二选一）

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    ipam:
      config:
        - subnet: 172.28.0.0/16
```

### 重启策略与资源限制

```yaml
services:
  app:
    restart: always              # always | on-failure | unless-stopped | no
    deploy:
      resources:
        limits:
          cpus: '0.50'          # CPU 限制
          memory: 512M           # 内存限制
        reservations:
          memory: 256M
```

## 常用示例

### LNMP 环境

```yaml
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d
      - ./www:/var/www/html
    depends_on:
      - php
    networks:
      - lnmp

  php:
    build: ./php
    volumes:
      - ./www:/var/www/html
    depends_on:
      - mysql
    networks:
      - lnmp

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: testdb
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - lnmp

networks:
  lnmp:

volumes:
  mysql_data:
```

### WordPress

```yaml
services:
  db:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: wordpress
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wordpress
      MYSQL_PASSWORD: wordpress
    volumes:
      - db_data:/var/lib/mysql

  wordpress:
    image: wordpress:latest
    restart: always
    ports:
      - "80:80"
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wordpress
      WORDPRESS_DB_PASSWORD: wordpress
      WORDPRESS_DB_NAME: wordpress
    depends_on:
      - db
    volumes:
      - wp_data:/var/www/html

volumes:
  db_data:
  wp_data:
```

## .env 文件

在项目根目录创建 `.env` 文件，定义环境变量：

```bash
# .env
MYSQL_ROOT_PASSWORD=root123
MYSQL_DATABASE=myapp
APP_PORT=8080
```

在 `compose.yml` 中引用：

```yaml
services:
  app:
    ports:
      - "${APP_PORT}:80"
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
```
