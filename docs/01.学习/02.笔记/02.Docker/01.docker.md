# Docker 基础

## 安装

```bash
# 更新软件包
sudo apt-get update

# 安装 curl 和 wget
apt install curl wget

# 安装 Docker
apt install docker.io

# 设置 Docker 自启动
systemctl enable docker
systemctl start docker
```

## 一般命令

| 命令 | 说明 |
|------|------|
| `docker ps` | 列出正在运行的容器 |
| `docker ps -a` | 列出所有容器 |
| `docker ps -s` | 列出正在运行的容器（带 CPU / 内存） |
| `docker images` | 列出所有镜像 |
| `docker exec -it <container> bash` | 连接到容器 |
| `docker logs <container>` | 显示容器的控制台日志 |
| `docker stop <container>` | 停止容器 |
| `docker restart <container>` | 重启容器 |
| `docker rm <container>` | 移除容器 |
| `docker port <container>` | 显示容器的端口映射 |
| `docker top <container>` | 列出进程 |
| `docker kill <container>` | 杀死容器 |

## 容器操作

### 创建和运行

```bash
# 在后台创建和运行容器
docker run -d -p 80:80 docker/getting-started

# 在前台创建并运行容器（退出不删除容器按 Ctrl+P+Q）
docker run -it --rm -p 8001:8080 --name my-nginx nginx
```

参数说明：

- `-d` - 以分离（后台）模式运行容器
- `-p 80:80` - 端口映射（宿主机端口:容器端口）
- `-it` - 交互式 bash 模式
- `--rm` - 容器终止运行后自动删除容器文件
- `--name` - 指定容器名称

### 启动和停止

```bash
docker start nginx-server        # 开始
docker stop nginx-server         # 停止
docker restart nginx-server      # 重启
docker pause nginx-server        # 暂停
docker unpause nginx-server      # 取消暂停
docker wait nginx-server         # 阻塞容器
docker kill nginx-server         # 发送 SIGKILL
docker attach nginx-server       # 连接到现有容器
```

### 查看容器

```bash
docker ps                        # 列出正在运行的容器
docker ps -a                     # 列出所有容器
docker logs nginx-server         # 容器日志
docker inspect nginx-server      # 检查容器
docker events nginx-server       # 容器事件
docker port nginx-server         # 公开端口
docker top nginx-server          # 运行进程
docker stats nginx-server        # 容器资源使用
docker diff nginx-server         # 列出对容器所做的更改
```

### 创建容器

```bash
docker create [options] IMAGE
  -a, --attach               # 附加标准输出/错误
  -i, --interactive          # 附加标准输入（交互式）
  -t, --tty                  # 伪终端
      --name NAME            # 命名容器
  -p, --publish 5000:5000    # 端口映射（主机:容器）
      --expose 5432          # 向容器公开端口
  -P, --publish-all          # 发布所有端口
      --link container:alias # 链接
  -v, --volume `pwd`:/app    # 挂载（需要绝对路径）
  -e, --env NAME=hello       # 环境变量
```

实例：

```bash
docker create --name my_redis --expose 6379 redis:3.0.2
```

### 操控容器

```bash
docker rename my-nginx nginx-server              # 重命名容器
docker rm nginx-server                           # 移除容器
docker update --cpu-shares 512 -m 300M nginx-server  # 更新容器
```

## 快捷键

| 快捷键 | 说明 |
|--------|------|
| `ctrl + c` | 关闭并删除容器（仅 `docker attach` 进入的容器） |
| `ctrl + d` | 保留容器并退出 |
| `ctrl + p + q` | 容器分离（保留容器但不退出） |

> 使用 `docker exec` 进入容器后，快捷键不会删除容器实例

## 数据卷

```bash
docker volume create 卷名           # 创建命名卷
docker volume inspect 卷名          # 查看命名卷详细信息
docker volume list                  # 列出所有卷
docker volume rm 卷名               # 删除卷
docker volume prune                 # 删除所有未使用的卷
```

## 批量清除

| 命令 | 说明 |
|------|------|
| `docker stop -f $(docker ps -a -q)` | 停止所有容器 |
| `docker rm -f $(docker ps -a -q)` | 删除所有容器 |
| `docker rmi -f $(docker images -q)` | 删除所有镜像 |
| `docker volume prune` | 删除所有未使用的 Docker Volume |
| `docker network prune` | 删除所有未使用的 Docker 网络 |
| `docker system prune` | 清理所有空闲或无关的资源 |
| `docker image prune` | 删除悬空的 Docker 镜像 |
| `docker container prune` | 删除所有未使用的 Docker 容器 |
