# Docker 笔记

## 安装

```bash
# 更新软件包
sudo apt-get update

# 安装 curl 和 wget
apt install curl wget

# 安装 Docker
apt install docker.io

# 设置 Docker 自启动
systemctl enable docker
systemctl start docker
```

## 容器操作

### 创建和运行

```bash
# 在后台创建和运行容器
docker run -d -p 80:80 docker/getting-started

# 在前台创建并运行容器（退出不删除容器按 Ctrl+P+Q）
docker run -it --rm -p 8001:8080 --name my-nginx nginx
```

参数说明：

- `-d` - 以分离（后台）模式运行容器
- `-p 80:80` - 端口映射（宿主机端口:容器端口）
- `-it` - 交互式 bash 模式
- `--rm` - 容器终止运行后自动删除容器文件
- `--name` - 指定容器名称

### 启动和停止

```bash
docker start nginx-server        # 开始
docker stop nginx-server         # 停止
docker restart nginx-server      # 重启
docker pause nginx-server        # 暂停
docker unpause nginx-server      # 取消暂停
docker wait nginx-server         # 阻塞容器
docker kill nginx-server         # 发送 SIGKILL
docker attach nginx-server       # 连接到现有容器
```

### 查看容器

```bash
docker ps                        # 列出正在运行的容器
docker ps -a                     # 列出所有容器
docker logs nginx-server         # 容器日志
docker inspect nginx-server      # 检查容器
docker events nginx-server       # 容器事件
docker port nginx-server         # 公开端口
docker top nginx-server          # 运行进程
docker stats nginx-server        # 容器资源使用
docker diff nginx-server         # 列出对容器所做的更改
```

### 创建容器

```bash
docker create [options] IMAGE
  -a, --attach               # 附加标准输出/错误
  -i, --interactive          # 附加标准输入（交互式）
  -t, --tty                  # 伪终端
      --name NAME            # 命名容器
  -p, --publish 5000:5000    # 端口映射（主机:容器）
      --expose 5432          # 向容器公开端口
  -P, --publish-all          # 发布所有端口
      --link container:alias # 链接
  -v, --volume `pwd`:/app    # 挂载（需要绝对路径）
  -e, --env NAME=hello       # 环境变量
```

实例：

```bash
docker create --name my_redis --expose 6379 redis:3.0.2
```

### 操控容器

```bash
docker rename my-nginx nginx-server              # 重命名容器
docker rm nginx-server                           # 移除容器
docker update --cpu-shares 512 -m 300M nginx-server  # 更新容器
```

## 镜像操作

### 基本操作

| 命令 | 说明 |
|------|------|

| `docker rmi nginx` | 删除镜像 |

| `docker history` | 显示镜像的历史 |

| `docker tag nginx eon01/nginx` | 标记镜像 |
| `docker push eon01/nginx` | 推送镜像 |

### 构建镜像

```bash
docker build .
docker build github.com/creack/docker-firefox
docker build - < Dockerfile
docker build - < context.tar.gz
docker build -t eon/nginx-server .
docker build -f myOtherDockerfile .
docker build --build-arg https_proxy=127.0.0.1:8088  # 使用 http 代理构建
curl example.com/remote/Dockerfile | docker build -f - .
docker save -o <保存路径>/myimage.tar myimage:latest  # 导出
docker load -i <路径>/myimage.tar                     # 导入
```

#### Dockerfile 入门示例

**1. 目录结构**

```
./
├── Dockerfile
└── app.py
```

**2. 编写业务代码 `app.py`**

```python
print("Hello Docker!")
```

**3. 编写 Dockerfile（最简版）**

```dockerfile
# 选用官方 python 基础镜像（自带 python3，无需额外安装）
FROM python:3.9-slim

# 复制当前目录所有文件到容器 /app 目录
COPY . /app

# 指定容器工作目录
WORKDIR /app

# 容器启动后执行命令
CMD ["python", "app.py"]
```

**4. 构建 & 运行**

```bash
# 构建镜像，镜像名:版本
docker build -t myapp:1.0 .

# 运行容器
docker run myapp:1.0
```

**运行结果：**

```
Hello Docker!
```

## 网络操作

### 创建网络

```bash
# 创建 overlay 类型网络：用于 Docker Swarm 集群，实现跨多台主机容器互通
docker network create -d overlay MyOverlayNetwork

# 创建 bridge 类型网络：单机自定义桥接网络，当前主机内容器互相通信，支持 DNS 解析
docker network create -d bridge MyBridgeNetwork

# 创建 host 类型网络：容器直接复用宿主机网络栈，无独立网络隔离，端口与宿主机共用
docker network create -d host MyHostNetwork

# 创建 none 类型网络：禁用容器所有网络功能，容器无网卡、无法访问内外网络，纯隔离环境
docker network create -d none MyNoneNetwork
```

自定义网络子网和网关：

```bash
docker network create -d overlay \
  --subnet=192.168.0.0/16 \
  --subnet=192.170.0.0/16 \
  --gateway=192.168.0.100 \
  --gateway=192.170.0.100 \
  --ip-range=192.168.1.0/24 \
  MyOverlayNetwork
```

### 网络操作

```bash
docker inspect MyContainer | grep Network           # 获取容器连接的网络
docker network inspect <network_name>               # 获取有关网络的信息
docker network connect <network_name> <container>   # 将容器连接到网络
docker run -it -d --network=<network_name> <container>  # 启动时连接到网络
docker network disconnect <network_name> <container>    # 断开容器与网络的连接
docker network rm <network_name>                       # 删除网络
docker network ls                                      # 列出网络
```

## Docker Compose

Docker Compose 是一个用于定义和运行多容器 Docker 应用的工具。通过一个 YAML 文件来配置应用的服务，然后使用一个命令创建并启动所有服务。

> Docker Compose V2 是 Docker 官方推出的新版本，以 Docker 插件形式安装，命令从 `docker-compose`（带连字符）改为 `docker compose`（无连字符）。

### 安装

```bash
# 创建 root 用户的插件目录
mkdir -p /root/.docker/cli-plugins

# 下到二进制文件
curl -SL https://v4.gh-proxy.org/https://github.com/docker/compose/releases/download/v2.29.2/docker-compose-linux-x86_64 \
  -o /root/.docker/cli-plugins/docker-compose

# 加执行权限
chmod +x /root/.docker/cli-plugins/docker-compose

# 验证
docker compose version
```

### 基本命令

| 命令 | 说明 |
|------|------|
| `docker compose up -d` | 后台启动所有服务 |
| `docker compose down` | 停止并移除容器、网络 |
| `docker compose ps` | 列出项目中的容器 |
| `docker compose logs` | 查看服务日志 |
| `docker compose build` | 构建或重新构建服务 |
| `docker compose exec <服务名> bash` | 进入运行中的容器 |
| `docker compose restart` | 重启所有服务 |
| `docker compose stop` | 停止所有服务 |
| `docker compose start` | 启动已停止的服务 |
| `docker compose rm` | 删除已停止的容器 |
| `docker compose pull` | 拉取服务依赖的镜像 |
| `docker compose config` | 验证并查看 compose 文件配置 |

### compose.yml 基础结构

```yaml
services:
  web:
    build: .                    # 使用当前目录的 Dockerfile 构建
    ports:
      - "80:80"                 # 端口映射
    volumes:
      - ./code:/app             # 挂载目录
    depends_on:
      - db                      # 依赖的其他服务
    environment:
      - DB_HOST=db              # 环境变量
    networks:
      - frontend                # 连接的网络

  db:
    image: mysql:8.0            # 使用镜像
    volumes:
      - db_data:/var/lib/mysql  # 数据卷
    environment:
      MYSQL_ROOT_PASSWORD: password
    networks:
      - frontend

networks:
  frontend:                     # 定义网络

volumes:
  db_data:                      # 定义数据卷
```

### 常用配置说明

#### 构建配置

```yaml
services:
  app:
    build:
      context: ./app            # Dockerfile 所在目录
      dockerfile: Dockerfile    # Dockerfile 文件名
      args:                     # 构建参数
        - APP_VERSION=1.0
      cache_from:
        - myimage:latest        # 缓存来源
```

#### 环境变量

```yaml
services:
  app:
    environment:
      - DB_HOST=localhost       # 方式一：列表形式
      - DB_PORT=3306
    env_file:                   # 方式二：从文件读取
      - .env
      - ./config/app.env
```

#### 端口映射

```yaml
services:
  web:
    ports:
      - "80:80"                 # host:container
      - "8080:8080"
      - "127.0.0.1:8081:8081"  # 绑定特定 IP
```

#### 数据卷

```yaml
services:
  db:
    volumes:
      - db_data:/var/lib/mysql           # 命名卷
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql  # 绑定挂载
      - /var/log:/logs:ro                # 只读挂载

volumes:
  db_data:                        # 声明命名卷
    driver: local
```

#### 网络配置

```yaml
services:
  web:
    networks:
      - frontend
      - backend
    network_mode: "host"         # 使用宿主机网络（二选一）

networks:
  frontend:
    driver: bridge
  backend:
    driver: bridge
    ipam:
      config:
        - subnet: 172.28.0.0/16
```

#### 重启策略与资源限制

```yaml
services:
  app:
    restart: always              # always | on-failure | unless-stopped | no
    deploy:
      resources:
        limits:
          cpus: '0.50'          # CPU 限制
          memory: 512M           # 内存限制
        reservations:
          memory: 256M
```

### .env 文件


```bash
docker run -itd -p 80:80 --restart=always --name Nginx \
-v $HOME/nginx_data/html:/usr/share/nginx/html \
-v $HOME/nginx_data/conf:/etc/nginx/conf.d \
```

| 参数 | 说明 |

## 修改数据存储路径

1. 停止 Docker 服务

```bash
sudo systemctl stop docker
```

2. 移动数据到新目录

```bash
sudo mv /var/lib/docker /new/path/docker
```

3. 修改配置文件 `/etc/docker/daemon.json`

```json
{ "data-root": "/new/path/docker" }
```

4. 重启 Docker 服务

```bash
sudo systemctl start docker
```
