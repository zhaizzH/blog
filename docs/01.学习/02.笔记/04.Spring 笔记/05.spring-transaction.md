# Spring 事务管理

## Spring 事务简介

- 事务作用：保障一系列数据库操作同成功同失败
- Spring 事务：在**数据层或业务层**保障事务
- 平台事务管理器：`PlatformTransactionManager`
- MyBatis 使用：`DataSourceTransactionManager`

![事务管理](https://img.zhaizz.top/file/blog-website/spring/1630243651541(1).png)

## 转账案例

```java
@Service
public class AccountServiceImpl implements AccountService {
    @Autowired
    private AccountDao accountDao;

    @Transactional    // 开启事务（一般写在接口上）
    public void transfer(String out, String in, Double money) {
        accountDao.outMoney(out, money);
        accountDao.inMoney(in, money);
    }
}
```

```java
// JdbcConfig 中配置事务管理器
@Bean
public PlatformTransactionManager transactionManager(DataSource dataSource){
    DataSourceTransactionManager tm = new DataSourceTransactionManager();
    tm.setDataSource(dataSource);
    return tm;
}
```

```java
@Configuration
@EnableTransactionManagement  // 开启注解式事务
public class SpringConfig { }
```

## 事务角色

![事务角色](https://img.zhaizz.top/file/blog-website/spring/1630249111055(1).png)

| 角色 | 说明 |
|------|------|
| 事务管理员 | 发起事务方，业务层开启事务的方法 |
| 事务协调员 | 加入事务方，数据层方法 |

## 事务属性

![事务属性](https://img.zhaizz.top/file/blog-website/spring/1630250069844(1).png)

| 属性 | 说明 |
|------|------|
| readOnly | 只读事务（true）/ 读写事务（false） |
| timeout | 超时时间（秒），-1 不设置 |
| rollbackFor | 指定异常回滚（默认只回滚 RuntimeException 和 Error） |
| noRollbackFor | 指定异常不回滚 |
| isolation | 隔离级别 |
| propagation | 事务传播行为 |

## 事务传播行为

![事务传播行为](https://img.zhaizz.top/file/blog-website/spring/1630253779575(1).png)

```java
@Transactional(propagation = Propagation.REQUIRES_NEW)  // 新事务
public void log(String out, String in, Double money) {
    logDao.log("转账操作由" + out + "到" + in);
}
```

| 传播行为 | 说明 |
|------|------|
| REQUIRED（默认） | 有事务则加入，无事务则新建 |
| REQUIRES_NEW | 无条件新建事务 |
| SUPPORTS | 有事务则加入，无事务则非事务运行 |
| NOT_SUPPORTED | 以非事务运行，挂起当前事务 |
| MANDATORY | 必须有事务，否则抛异常 |
| NEVER | 不能有事务，否则抛异常 |
| NESTED | 嵌套事务（Savepoint） |
