## 核心概念

### IOC（控制反转）

使用对象时，由主动 `new` 产生对象转换为由**外部**提供对象，对象创建控制权由程序转移到外部。

![IOC 概念](https://img.zhaizz.top/file/blog-website/spring/1629724206002.png)

- Spring 提供 **IOC 容器**充当"外部"
- 容器中存放的对象统称为 **Bean**
- 被管理的对象在 IOC 容器中统称为 Bean

### DI（依赖注入）

在容器中建立 Bean 与 Bean 之间的依赖关系的整个过程。

![DI 概念](https://img.zhaizz.top/file/blog-website/spring/1629735078619.png)

**目标**：充分解耦

1. 使用 IOC 容器管理 Bean（IOC）
2. 在 IOC 容器内将有依赖关系的 Bean 进行关系绑定（DI）
3. 使用对象时直接从 IOC 容器中获取，且已绑定所有依赖关系

## 入门案例

### IOC 入门

```xml
<bean id="bookDao" class="com.itheima.dao.impl.BookDaoImpl"/>
<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl"/>
```

```java
ApplicationContext ctx = new ClassPathXmlApplicationContext("applicationContext.xml");
BookService bookService = (BookService) ctx.getBean("bookService");
bookService.save();
```

### DI 入门（Setter 注入）

```xml
<bean id="bookService" class="com.itheima.service.impl.BookServiceImpl">
    <property name="bookDao" ref="bookDao"/>
</bean>
```
