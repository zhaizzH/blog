# MyBatisPlus DQL 查询控制

## 条件查询

MP 将书写复杂的 SQL 查询条件进行了封装，使用编程的形式完成查询条件的组合。

### 条件查询的三种方式

#### 方式一：QueryWrapper

```java
@Test
void testGetAll(){
    QueryWrapper qw = new QueryWrapper();
    qw.lt("age",18);
    List<User> userList = userDao.selectList(qw);
    System.out.println(userList);
}
```

- `lt`：小于（<），最终 SQL 为 `SELECT id,name,password,age,tel FROM user WHERE (age < ?)`
- 缺点：条件字段使用字符串，容易写错

#### 方式二：QueryWrapper + Lambda

```java
@Test
void testGetAll(){
    QueryWrapper<User> qw = new QueryWrapper<User>();
    qw.lambda().lt(User::getAge, 10);
    List<User> userList = userDao.selectList(qw);
    System.out.println(userList);
}
```

- 使用 `User::getAge` lambda 表达式，避免字段名写错
- 缺点：需要多一层 `lambda()` 调用

#### 方式三：LambdaQueryWrapper（推荐）

```java
@Test
void testGetAll(){
    LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
    lqw.lt(User::getAge, 10);
    List<User> userList = userDao.selectList(lqw);
    System.out.println(userList);
}
```

**注意：** 构建 LambdaQueryWrapper 的时候泛型不能省。

### 多条件构建

**AND 条件（默认）：**

> 需求：查询年龄在 10 岁到 30 岁之间的用户信息

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.lt(User::getAge, 30);
lqw.gt(User::getAge, 10);
List<User> userList = userDao.selectList(lqw);
```

支持链式编程：

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.lt(User::getAge, 30).gt(User::getAge, 10);
```

最终 SQL：`SELECT id,name,password,age,tel FROM user WHERE (age < ? AND age > ?)`

**OR 条件：**

> 需求：查询年龄小于 10 或年龄大于 30 的数据

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.lt(User::getAge, 10).or().gt(User::getAge, 30);
```

最终 SQL：`SELECT id,name,password,age,tel FROM user WHERE (age < ? OR age > ?)`

### null 判断

在做条件查询时，用户可能只输入部分条件，如只输入价格下限不输入上限。如果不处理 null，会出现 `price > 8000 AND price < null` 的问题。

**方案：** 使用查询条件方法的 `condition` 参数进行 null 判断：

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.lt(null!=uq.getAge2(), User::getAge, uq.getAge2());
lqw.gt(null!=uq.getAge(), User::getAge, uq.getAge());
```

`condition` 为 boolean 类型，返回 true 则添加条件，返回 false 则不添加条件。

## 查询投影

### 查询指定字段

不查询所有字段，只查询出指定内容的数据：

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.select(User::getId, User::getName, User::getAge);
List<User> userList = userDao.selectList(lqw);
```

最终 SQL：`SELECT id,name,age FROM user`

如果不使用 lambda，需要手动指定字段：

```java
QueryWrapper<User> lqw = new QueryWrapper<User>();
lqw.select("id","name","age","tel");
List<User> userList = userDao.selectList(lqw);
```

### 聚合查询

```java
QueryWrapper<User> lqw = new QueryWrapper<User>();
//lqw.select("count(*) as count");     // SELECT count(*) as count FROM user
//lqw.select("max(age) as maxAge");    // SELECT max(age) as maxAge FROM user
//lqw.select("min(age) as minAge");    // SELECT min(age) as minAge FROM user
//lqw.select("sum(age) as sumAge");    // SELECT sum(age) as sumAge FROM user
lqw.select("avg(age) as avgAge");       // SELECT avg(age) as avgAge FROM user
List<Map<String, Object>> userList = userDao.selectMaps(lqw);
```

聚合函数起了别名，方便后期获取数据。

### 分组查询

```java
QueryWrapper<User> lqw = new QueryWrapper<User>();
lqw.select("count(*) as count,tel");
lqw.groupBy("tel");
List<Map<String, Object>> list = userDao.selectMaps(lqw);
```

最终 SQL：`SELECT count(*) as count,tel FROM user GROUP BY tel`

**注意：** 聚合与分组查询无法使用 lambda 表达式，如果 MP 实现不了，可以直接在 DAO 接口中使用 MyBatis 的方式实现。

## 查询条件

### 等值查询

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.eq(User::getName, "Jerry").eq(User::getPassword, "jerry");
User loginUser = userDao.selectOne(lqw);
```

- `eq()`：相当于 `=`
- `selectOne`：查询结果为单个

### 范围查询

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.between(User::getAge, 10, 30);
List<User> userList = userDao.selectList(lqw);
```

范围查询方法：

| 方法 | 说明 |
|------|------|
| `gt()` | 大于（>） |
| `ge()` | 大于等于（>=） |
| `lt()` | 小于（<） |
| `le()` | 小于等于（<=） |
| `between()` | BETWEEN ? AND ? |

### 模糊查询

```java
LambdaQueryWrapper<User> lqw = new LambdaQueryWrapper<User>();
lqw.likeLeft(User::getName, "J");
List<User> userList = userDao.selectList(lqw);
```

模糊查询方法：

| 方法 | 说明 | 示例 |
|------|------|------|
| `like()` | 前后加百分号 | %J% |
| `likeLeft()` | 前面加百分号 | %J |
| `likeRight()` | 后面加百分号 | J% |

### 排序查询

```java
LambdaQueryWrapper<User> lwq = new LambdaQueryWrapper<>();
/**
 * condition ：条件，返回boolean，当condition为true，进行排序，如果为false，则不排序
 * isAsc:是否为升序，true为升序，false为降序
 * columns：需要操作的列
 */
lwq.orderBy(true, false, User::getId);
userDao.selectList(lwq);
```

排序方法：

| 方法 | 说明 |
|------|------|
| `orderBy(condition, isAsc, columns)` | 条件排序，支持多字段 |
| `orderByAsc(columns)` | 按指定字段升序 |
| `orderByDesc(columns)` | 按指定字段降序 |

## 映射匹配兼容性

表字段与模型类属性默认通过名称对应，当出现不一致时需要特殊处理。

### @TableField 注解

| 名称 | @TableField |
|------|-------------|
| 类型 | 属性注解 |
| 位置 | 模型类属性定义上方 |
| 作用 | 设置当前属性对应的数据库表中的字段关系 |
| 相关属性 | `value`（默认）：设置数据库表字段名称<br/>`exist`：设置属性在数据库表字段中是否存在，默认为 true<br/>`select`：设置属性是否参与查询，默认为 true |

**问题 1：表字段与编码属性设计不同步**

使用 `@TableField(value="pwd")` 建立映射关系：

```java
@TableField(value="pwd")
private String password;
```

**问题 2：编码中添加了数据库中未定义的属性**

使用 `@TableField(exist=false)` 排除字段：

```java
@TableField(exist=false)
private Integer online;
```

**问题 3：默认查询开放了更多字段查看权限**

使用 `@TableField(select=false)` 隐藏敏感字段：

```java
@TableField(value="pwd", select=false)
private String password;
```

### @TableName 注解

| 名称 | @TableName |
|------|------------|
| 类型 | 类注解 |
| 位置 | 模型类定义上方 |
| 作用 | 设置当前类对应于数据库表关系 |
| 相关属性 | `value`（默认）：设置数据库表名称 |

当表名与模型类名不一致时使用：

```java
@Data
@TableName("tbl_user")
public class User {
    private Long id;
    private String name;
    // ...
}
```
